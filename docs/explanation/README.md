# Explanation

## Purpose

Store conceptual and architecture-facing documentation that helps contributors understand how Dictum is organized and how to read the system.

## Documents

- [Product Scope and Boundaries](./product-scope-and-boundaries.md)
- [Backend Layering and Service Boundaries](./backend-layering-and-service-boundaries.md)

## Scope Boundaries

- Explanation documents describe the architectural model, vocabulary, and conceptual structure of the system.
- Explanation documents should read as stable system understanding, not as proposals or implementation checklists.
- Prefer concept-shaped sections such as boundaries, ownership, flows, or models over decision-shaped sections such as problem, choice, or consequences.
- Reference material belongs in [Reference](../reference/README.md).
- Chosen tradeoffs, migration decisions, and option comparisons belong in [Decision Records](../decisions/README.md).
